Motorcycle Iteration 2: Schelling's Dynamic Models of Segregation

Project Overview

Welcome to Motorcycle Iteration 2, the latest iteration of Schelling's Dynamic Models of Segregation. This version includes a range of exciting improvements designed to enhance the realism and complexity of agent-based simulations.

This project integrates 50 agents who make decisions based on a variety of demographic factors such as age, salary, career, and household status. Each agent is now paired with a digital twin—a simulated persona that offers a deeper understanding of their motivations and decisions within the simulation. This digital twin provides feedback explaining the agent's actions as they seek neighborhoods that meet their personal preferences.

  • Digital twin logic: AI-powered personas that explain agents' movement decisions.
  • Demographic profiles: Agents possess unique traits such as age, career, and household composition.
  • Improved visualizations: Larger grids and enhanced movement tracking.
  • No explicit tipping point logic: Emergent behaviors arise organically from agent interactions.

Key Features

The goal of Motorcycle Iteration 2 is to explore how individual preferences, in combination with socioeconomic factors, can lead to larger societal patterns of segregation. This version provides new insights into Schelling’s foundational theories by introducing modern computational techniques such as digital twin agents and demographic-driven decision-making.

  • Demographic Profiles: Agents now include details such as age, career, salary, and household data, influencing their behavior.
  • Digital Twin Surveys: Each agent's digital twin provides real-time feedback on their decisions.
  • Simulated Neighborhood Dynamics: Agents assess their satisfaction with their neighborhood and relocate if unsatisfied.
  • Grid Visualization: View initial and final grid conditions to visualize the segregation dynamics.

Sample Code Snippet

Below is a snippet of the core logic for agent movement and decision-making in the simulation:


class Agent:
    def __init__(self, id, group, preference, current_location, age, salary, career, household_status, household_size):
        self.id = id
        self.group = group  
        self.preference = preference  
        self.current_location = current_location  
        self.age = age
        self.salary = salary
        self.career = career
        self.household_status = household_status
        self.household_size = household_size

    def is_satisfied(self, neighbors):
        return self.calculate_similarity(neighbors) >= self.preference
